# QuizNote (Personal Quiz & Wrong Answer App)

## 📌 프로젝트 개요
수익 창출 목적이 아닌, **나의 개인 학습(문제 풀이 및 오답 노트)을 위한 iOS 앱**.
단순한 문제 풀이뿐만 아니라 **"오답 경험 설계"**에 집중하여, 틀린 문제를 반복 학습할 수 있는 구조를 만듭니다.

## 🎯 수익형이 아닌 "개인용"에 맞춘 핵심 변경/추가 사항
1. **문제 데이터 대량 입력(Import) 기능 추가**
   - 모바일 기기에서 일일이 문제를 타이핑하는 것은 매우 비효율적입니다.
   - **해결책**: CSV 또는 JSON 파일로 문제를 작성한 뒤, iOS의 **"파일(Files) 앱" 또는 iCloud Drive를 통해 한 번에 Import** 하는 기능을 1차 스펙에 추가합니다.
2. **에빙하우스 망각곡선 / 단순 반복 로직 추가**
   - 단순 오답 저장이 아닌, **"3번 틀린 문제 모아보기", "어제 틀린 문제 다시 풀기"** 등 실질적인 복습 필터를 강화합니다.
3. **불필요한 기능 제거**
   - Firebase 연동, 로그인, 프리미엄 결제 모델, 유저 데이터 수집 등은 모두 제외하고 오프라인(로컬 Core Data) 중심의 빠르고 가벼운 앱으로 유지합니다.
4. **로컬 배포 최적화**
   - TestFlight 배포를 거치지 않고, 필요 시 로컬 빌드로 직접 기기에 올리거나, 무료 개발자 계정의 7일 갱신 제한을 피할 간단한 워크플로우를 고려합니다 (또는 본인 계정에 TestFlight로 올리는 용도).

## 🗂 파일 구조 플랜 (SwiftUI + CoreData)
- `Models/`: Question, WrongAnswer (CoreData)
- `Views/`: HomeView, QuizView, ResultView, WrongNoteView, ImportView
- `ViewModels/`: QuizViewModel, CoreDataManager
- `Data/`: SampleQuestions.json (초기 테스트용)

## 🚀 2주 개발 로드맵 (수정판)
- **1~3일차**: 프로젝트 세팅, UI 와이어프레임 구현 (홈, 퀴즈, 오답노트), JSON/CSV 파싱 로직 및 Import 기능(핵심) 구현
- **4~6일차**: 문제 풀이 뷰 + 채점 로직 + 해설 보기 기능 완료
- **7~9일차**: Core Data 설계 및 오답 저장/불러오기 기능 연결
- **10~11일차**: 오답노트 전용 뷰 (다시 풀기, 필터링) 기능 구현
- **12~14일차**: 햅틱 피드백(진동), UX/UI 다듬기, 본인 폰에 빌드 및 실사용 테스트
