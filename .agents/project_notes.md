## 🏛 프로젝트 구조 및 핵심 로직

### 1. 데이터 파이프라인
- **Source**: `Data/examples/*.pdf` (원본 기출문제)
- **Extraction**: `scripts/mass_extract.py` (Python + pdfplumber)
- **Storage**: `src/Data/exams/*.json` (추출된 정제 데이터)
- **Service**: `src/services/dataService.ts` (앱 내 데이터 공급 및 검증)

### 2. 주요 아키텍처 결정
- **정답 인덱스**: 모든 데이터는 **1-indexed** (1~4 또는 1~5)로 통일. `0`은 데이터 누락을 의미함.
- **데이터 검증**: `DataService`에서 런타임에 정답이 없는 문제를 필터링하여 사용자 경험 보호.
- **오답 관리**: SQLite(`DbService`)를 통해 틀린 문제 ID와 사용자 선택지를 저장하여 오답 노트 구성.

## ⚠️ 기술 부채 및 주의 사항
- **PDF 파싱 한계**: 일부 특이 양식 PDF에서 선택지가 질문 텍스트 끝에 붙어 나오는 현상이 잔존함.
- **해설 부재**: 현재 대부분의 과거 데이터가 "해설은 준비 중입니다." 상태임. 향후 외부 API 연동 또는 수동 보강 필요.
- **NFD/NFC 이슈**: Mac 환경에서의 파일명 인코딩 문제로 인해 `DataService` 임포트 시 파일명 일치 여부 주의 필요.
